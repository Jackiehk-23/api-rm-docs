## Step by Step

### Step 1 — Sort JSON Alphabetically

Sort **all** keys alphabetically, including nested objects, then remove all spaces and newlines so the JSON is compact.

**Method:** <HttpMethodBadge method="POST" />

<Admonition type="important">
Both the **data object** and every **nested object** must be sorted. Before encoding, replace special characters:

| Character | Replace With |
|-----------|-------------|
| `<` | `\u003c` |
| `>` | `\u003e` |
| `&` | `\u0026` |
</Admonition>

<Admonition type="note">
The request body depends on which API endpoint you are calling. The body below is just an **EXAMPLE**.
</Admonition>

### Step 3 — Construct the Signing String

Combine the encoded body with the request parameters, joined by `&` in **alphabetical** order.

<Admonition type="important">
- If the body is empty, the `data` parameter can be skipped.
- If you are verifying a callback signature, the `requestUrl` can be skipped.
</Admonition>

### Step 4 — Sign with Your Private Key

Sign the signing string using **RSA SHA256** with your private key. Base64-encode the result.

<Admonition type="important">
Make sure the matching **public key** has been uploaded to the **RM Merchant Portal**, or verification will fail.
</Admonition>

<Tabs groupId="lang">
<TabItem value="node" label="Node.js" default>

<CodeBlock language="javascript" filename="Node.js">
{`const crypto = require('crypto');
const fs = require('fs');

// Load your RSA private key from a secure location (never hardcode it)
const privateKeyPem = fs.readFileSync(process.env.RM_PRIVATE_KEY_PATH, 'utf8');
const url = 'https://sb-open.revenuemonster.my/v3/payment/online';

const body = {
  order: { title: 'hello', detail: '', additionalData: 'world',
           amount: 10, currencyType: 'MYR', id: '7211' },
  customer: { userId: '13245876', email: '' },
  method: [],
  type: 'WEB_PAYMENT',
  storeId: '1608123035564538121',
  redirectUrl: 'https://revenuemonster.my',
  notifyUrl: 'https://dev-rm-api.ap.ngrok.io',
  layoutVersion: 'v3',
};

// 1. Sort keys recursively, then compact + escape special characters
function sortObject(obj) {
  if (Array.isArray(obj)) return obj.map(sortObject);
  if (obj && typeof obj === 'object') {
    return Object.keys(obj).sort().reduce((acc, k) => {
      acc[k] = sortObject(obj[k]);
      return acc;
    }, {});
  }
  return obj;
}
const compact = JSON.stringify(sortObject(body))
  .replace(/</g, '\\\\u003c')
  .replace(/>/g, '\\u003e')
  .replace(/&/g, '\\u0026');

// 2. Base64 encode
const data = Buffer.from(compact).toString('base64');

// 3. Build the signing string (params in alphabetical order)
const nonceStr = crypto.randomBytes(16).toString('hex');
const timestamp = Math.floor(Date.now() / 1000).toString();
const signString =
  'data=' + data +
  '&method=post' +
  '&nonceStr=' + nonceStr +
  '&requestUrl=' + url +
  '&signType=sha256' +
  '&timestamp=' + timestamp;

// 4. Sign with RSA SHA256 private key
const signature = crypto.createSign('RSA-SHA256')
  .update(signString)
  .sign(privateKeyPem, 'base64');

// Send these headers with your request:
//   X-Signature:  'sha256 ' + signature
//   X-Nonce-Str:  nonceStr
//   X-Timestamp:  timestamp`}
</CodeBlock>

<CodeBlock language="python" filename="Python">
{`import json, base64, time, os, secrets
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import padding

# Load your RSA private key from a secure location (never hardcode it)
with open(os.environ["RM_PRIVATE_KEY_PATH"], "rb") as f:
    private_key = serialization.load_pem_private_key(f.read(), password=None)

url = "https://sb-open.revenuemonster.my/v3/payment/online"
body = {
    "order": {"title": "hello", "detail": "", "additionalData": "world",
              "amount": 10, "currencyType": "MYR", "id": "7211"},
    "customer": {"userId": "13245876", "email": ""},
    "method": [],
    "type": "WEB_PAYMENT",
    "storeId": "1608123035564538121",
    "redirectUrl": "https://revenuemonster.my",
    "notifyUrl": "https://dev-rm-api.ap.ngrok.io",
    "layoutVersion": "v3",
}

def sort_obj(o):
    if isinstance(o, dict):
        return {k: sort_obj(o[k]) for k in sorted(o)}
    if isinstance(o, list):
        return [sort_obj(i) for i in o]
    return o

# 1. Sort + compact + escape
compact = json.dumps(sort_obj(body), separators=(",", ":"), ensure_ascii=False)
compact = compact.replace("<", "\\\\u003c").replace(">", "\\\\u003e").replace("&", "\\\\u0026")

# 2. Base64 encode
data = base64.b64encode(compact.encode()).decode()

# 3. Build the signing string
nonce = secrets.token_hex(16)
timestamp = str(int(time.time()))
sign_string = (
    f"data={data}&method=post&nonceStr={nonce}"
    f"&requestUrl={url}&signType=sha256&timestamp={timestamp}"
)

# 4. Sign with RSA SHA256 private key
sig = private_key.sign(sign_string.encode(), padding.PKCS1v15(), hashes.SHA256())
signature = base64.b64encode(sig).decode()
# Send header  ->  X-Signature: f"sha256 {signature}\"`}

## Common Mistakes

<div className="sig-mistakes">
  <div className="sig-mistake">
    <strong>❌ JSON keys not sorted</strong>
    <p>Sort alphabetically including nested objects. Check every level.</p>
  </div>
  <div className="sig-mistake">
    <strong>❌ Extra spaces in JSON</strong>
    <p>Make it compact: <code>{`{"a":1}`}</code> not <code>{`{"a": 1}`}</code>. Remove all whitespace.</p>
  </div>
  <div className="sig-mistake">
    <strong>❌ Forgot to escape special characters</strong>
    <p>Replace <code>&lt;</code> → <code>{'\\u003c'}</code>, <code>&gt;</code> → <code>{'\\u003e'}</code>, <code>&amp;</code> → <code>{'\\u0026'}</code> before encoding.</p>
  </div>
  <div className="sig-mistake">
    <strong>❌ Timestamp too old</strong>
    <p>Must be within <strong>120 seconds</strong> of server UTC time. Sync your clock.</p>
  </div>
  <div className="sig-mistake">
    <strong>❌ Reused nonce</strong>
    <p>Generate a new nonce for each request. Don't reuse within 120 seconds.</p>
  </div>
  <div className="sig-mistake">
    <strong>❌ Wrong private key</strong>
    <p>Verify your private key matches the public key uploaded to the portal.</p>
  </div>
</div>

### Common Causes

| Cause | Fix |
|---|---|
| Private key doesn't match the public key uploaded to the Merchant Portal | Re-upload the correct public key |
| JSON keys not sorted alphabetically | Check nested objects too |
| JSON body still contains spaces or newlines before Base64 encoding | Make the JSON compact |
| Special characters (`<`, `>`, `&`) not replaced before encoding | Replace with `\u003c`, `\u003e`, `\u0026` |
| `X-Timestamp` is more than 120 seconds away from server UTC time | Sync your system clock |
| `X-Nonce-Str` was reused within a 120-second window | Generate a new nonce for each request |
| Amount is in cents — `"amount": 100` = RM 1.00 | Multiply the amount by 100 |
